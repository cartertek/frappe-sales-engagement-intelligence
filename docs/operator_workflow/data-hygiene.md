# Data Hygiene

Data hygiene utilities identify duplicate prospects, duplicate signals, missing source arenas, missing evidence, missing signal source URLs, inferred qualifying signal issues, and normalized-domain gaps.

Run dry-run or review-style utilities before making bulk changes. Import reset-to-draft is manager-controlled and should be used only when an import batch needs safe correction and reprocessing.

Data hygiene utilities should preserve Do Not Contact and Rejected protections. They do not create CRM records or send outreach.
